Cloudron Email Server

No DNS provider is setup. Displayed DNS records will have to be setup manually.
Cloudron will setup Email related DNS records automatically. If this domain is already configured to handle email with some other provider, it will overwrite those records.

Disabling Cloudron Email later will not put the old records back.

Status of DNS Records will show an error while DNS is propagating (~5 minutes).

Email

IMAP and SMTP Server

Cloudron has a built-in email server that allows users to send and receive email for your domain.

Mail server settings for email clients

Incoming Mail (IMAP)
Server: my.{{config.fqdn}}
Port: 993 (TLS)

Outgoing Mail (SMTP)
Server: my.{{config.fqdn}}
Port: 587 (STARTTLS)

ManageSieve
Server: my.{{config.fqdn}}
Port: 4190 (TLS)

All the servers require your Cloudron credentials for authentication.


This feature requires the Cloudron to be on custom domain.
